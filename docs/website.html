The send pipeline

Click send → delivered → tracked.

One button kicks off a pipeline built for at-least-once delivery and idempotent processing end to end.

01

Trigger

The API persists the run and produces a single dispatch command — the request returns instantly.

02

Fan out

A dispatcher pages the segment and enqueues one job per contact onto Kafka.

03

Throttle & send

Workers check the Redis rate limit and suppression cache, then deliver via SendGrid.

04

Track

Delivery events flow back through webhooks and into the events stream, idempotently.

05

Analyze

Live counters update the dashboard; the campaign closes when every contact is settled.
