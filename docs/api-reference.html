BulkSend · HTTP Surface

REST API Reference

Endpoint map across the five services. All app endpoints sit behind /api/v1; tracking & webhook endpoints are public and served on lean, separately-scaled routes. Tenant scope is derived from the access token — never from a path param.

§

Conventions

Auth

Short-lived JWT access token in Authorization: Bearer; rotating refresh token in an httpOnly cookie. The workspaceId claim scopes every request.

Pagination

Cursor or page params ?page / ?limit (max 100). List responses wrap as { data:[…], pagination:{ page, limit, total } }.

Errors

Consistent envelope { error:{ code, message, details } } with standard HTTP status. 422 for validation, 409 for conflicts.

Idempotency

State-changing sends accept an Idempotency-Key header; the server dedupes retries so a double-clicked “Send” fires once.

Publicprovider-facing · SendGrid
EndpointRequestResponseLayerMiddleware
POST/webhooks/sendgrid
 headers X-Twilio-Email-Event-Webhook-Signature / -Timestamp
body Event[] { sg_event_id, sg_message_id, event, email, timestamp, reason?, url? }		 200 — ack immediately (empty) Webhookverify → normalize → produce email.events
rawBodyverifyEd25519
NOTE

Verify before trust: validate SendGrid's Ed25519 signature against the raw body, then ack 2xx fast and hand the batch to Kafka — never block the response on DB writes. Each sg_event_id dedupes downstream (see events.provider_event_id), so SendGrid's at-least-once retries are safe.
